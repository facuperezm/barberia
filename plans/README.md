in the audit conversation and the file references below.)

## Findings considered and rejected

- **Zod 3 → 4 migration**: already evaluated and settled in
  `pnpm-workspace.yaml` comments (better-auth peer constraint; zod 3.25 ships
  the v4 API). Not worth doing.
- **Tailwind 3 → 4**: supported version, no security cutoff, large blast
  radius (CSS-first config, shadcn/animate compat), no forcing function.
  Re-evaluate only with a concrete need.
- **@types/node 20 → 22**: trivial, do opportunistically alongside any other
  dependency PR; not plan-worthy.
- **Formatter/pre-commit hooks + oxlint config**: CI already gates
  typecheck+lint+test; marginal value at current repo size.
- **Rate limiter failing open on DB errors**: deliberate, documented in-code
  (`src/lib/rate-limit.ts`) — availability over strictness for booking.
- **BetterAuth global rate-limit config**: follow-up noted in plan 006; the
  per-email throttle covers the concrete abuse case.
- **In-memory dev magic-token capture, open self-serve signup, dev-only
  instant sign-in, legacy dual appointment columns, manual-WhatsApp v1**: all
  verified as documented decisions with sound implementations; not findings.
